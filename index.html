Native HTML Accordion (Details/Summary)

The <details> and <summary> elements provide built-in expandable content functionality.

What is the <details> element?

The <details> element is a native HTML disclosure widget that can toggle between showing and hiding additional content. It's part of the HTML5 specification and works without any JavaScript.

According to MDN Web Docs, the details element creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state.

Browser Support & Accessibility

Modern browser support for <details> is excellent:

  • Chrome 12+ (2011)
  • Firefox 49+ (2016)
  • Safari 6+ (2012)
  • Edge 79+ (2020)

The element is fully accessible with keyboard navigation using the Enter or Space keys to toggle.

Advanced Styling Options

You can extensively customize the appearance using CSS:

  • Custom disclosure markers with summary::marker
  • Animated transitions for smooth opening/closing
  • Different states with [open] attribute selector
  • Nested details for hierarchical content
Real-World Use Cases

The <details> element is perfect for:

  • FAQ sections - Toggle questions and answers
  • Documentation - Collapsible code examples
  • Settings panels - Show/hide advanced options
  • Mobile menus - Expandable navigation
  • Form sections - Progressive disclosure

CSS-Only Modal Dialog

How It Works

This modal uses a hidden checkbox and CSS selectors to toggle visibility:

  • A hidden <input type="checkbox"> controls the state
  • A <label> triggers the checkbox when clicked
  • CSS :checked pseudo-class shows/hides the modal
  • The ~ sibling combinator targets the overlay

Advantages

  • No JavaScript dependencies
  • Lightweight and fast
  • Works with CSS disabled (graceful degradation)
  • Fully customizable with CSS
